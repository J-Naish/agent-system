# クリエイティブ基礎リファレンス

---

## 1. Aesthetic Score（全フォーマット共通）

2025年にX導入。すべての広告のクリエイティブ品質をAIで評価し、CPMに直接影響する。高スコアで最大20% CPM割引。

### 必須ルール

| ルール | 詳細 |
|--------|------|
| ハッシュタグ禁止 | 2025年6月27日以降、広告内でのハッシュタグ使用は完全禁止 |
| 絵文字は最大1つ | 日本・韓国は例外（より柔軟）、デフォルトは最大1つ |
| 本文にURLを入れない | Webサイトカードを使用する（本文中のURLは品質スコアを下げる） |
| 文法・句読点 | 誤字脱字、不自然な表現はペナルティ対象 |
| クリーンなデザイン | ごちゃついたビジュアル、フラッシュ効果、不要なテキストオーバーレイを避ける |

### 高スコアの特徴

- エレガントでミニマリストなデザイン
- 簡潔で洗練されたコピー
- オーガニック投稿のような自然な見た目
- 高品質なビジュアル
- 広告とLPのメッセージ一貫性

### 低スコアの特徴

- 過剰なテキストオーバーレイ
- 派手なエフェクト・大音量のオーディオ
- 使い古された緊急性表現
- ぼやけた低解像度画像
- いかにも「広告」なデザイン

## 2. クリエイティブ基本ルール

- 3つ以上のフォーマットを組み合わせる：認知度 +20%、購買意向 +7%
- 1広告グループにつき5つ以上のユニーククリエイティブ
- 1広告 = 1メッセージ
- すべてのビジュアルにロゴ + ブランドカラーを含める
- セールスキャンペーン：AI最適化に最低4〜6週間必要
- 判断を下す前に4〜6週間のデータを待つ

### 動画ルール

- 最初の3秒 = フック（最も記憶に残る部分）
- 15秒以内を推奨
- キャプション・字幕は必須
- ロゴは常に左上
- 縦型動画：音声ONを前提にデザインする

## 3. コピーライティングフレームワーク

X広告で効果的な4つのフレームワーク：

### 3-1. AIDA（Attention → Interest → Desire → Action）

- 最適な用途：製品ローンチ、機能発表
- 注意を引き、興味を持たせ、欲求を刺激し、行動を促す

### 3-2. PAS（Problem → Agitate → Solve）

- 最適な用途：SaaS、サービス、B2B
- 問題を提示し、痛みを増幅させ、解決策を示す

### 3-3. BAB（Before → After → Bridge）

- 最適な用途：ビフォーアフターが明確な商品（ダイエット、スキルアップ）
- 現状を示し、理想の状態を見せ、そこに至る橋渡しを提案する

### 3-4. Question → Answer

- 最適な用途：Xの会話文化に最もネイティブなフォーマット
- 問いかけで関心を引き、回答で価値を提供する

### X広告コピー vs 他プラットフォーム比較

| 要素 | X | Meta | Google |
|------|---|------|--------|
| 最適な長さ | 50〜100文字 | 125文字（推奨） | 見出し30 + 説明文90文字 |
| トーン | ダイレクト、主張がある、会話的 | パーソナル、感情的 | 検索意図に合致、キーワード重視 |
| コンテキスト | ニュースフィード、リアルタイム | パーソナルフィード | 検索 |
| ハッシュタグ | 禁止（2025年6月〜） | 任意 | N/A |
| 絵文字 | 最大1つ | 複数OK | 限定的 |
| CTA | ソフト、ネイティブな感覚 | やや強めもOK | ダイレクト、強め |

### コピータイプ別効果

| タイプ | 効果 | 備考 |
|--------|------|------|
| データ・統計 | 高い信頼性、CTR向上 | %表記は金額よりも効果的 |
| 質問 | 会話を生み、エンゲージメント向上 | X文化にマッチ |
| 社会的証明 | コンバージョン +34%（テスティモニアル） | 具体的な数字があるとより効果的 |
| 緊急性・希少性 | FOMOが行動を促す | Aesthetic Scoreが過剰使用にペナルティ |
| ユーモア | シェア率が高く拡散しやすい | ブランドトーンと合致させる必要あり |

## 4. CTA設計

### 目的別推奨CTA

| 目的 | 推奨CTA | 理由 |
|------|---------|------|
| Webサイトトラフィック | "Learn More" "詳しくはこちら" | ソフトCTA = より高いCTR |
| セールス/EC | "Shop Now" "今すぐ購入" | 高意向ユーザー向けにダイレクトCTA |
| アプリインストール | "Install" "ダウンロード" | 明確なアクション指示 |
| リードジェネレーション | "無料で試す" "資料をダウンロード" | 無料の価値提供でハードルを下げる |
| エンゲージメント | "あなたの意見は？" "教えてください" | 会話を促すソフトCTA |
| 認知 | "Discover" "もっと見る" | 発見・探索の言語 |

### ソフトCTA vs ハードCTA

| タイプ | 例 | 最適な場面 |
|--------|---|-----------|
| ソフト | もっと見る、詳しくはこちら、探す | TOFU（認知）、新規ユーザー、Aesthetic Scoreを重視する場合 |
| ハード | 今すぐ購入、ダウンロード、申し込む | BOFU（購買）、リターゲティング、明確なオファー |

### CTA設計ルール

- 1広告 = 1CTA（複数CTAを入れない）
- Webサイトカードのボタン（最大24文字）とコピーのCTAを揃える
- モバイル（80%以上）→ 大きなタップエリア
- コピーのCTAとボタンのCTAで同じ言葉を使わない（冗長性を避ける）
  - 例：コピー「新機能をチェックしませんか」+ ボタン「詳しくはこちら」

## 5. ランディングページ最適化

### メッセージマッチ

- 広告のビジュアル・コピーとLPのファーストビューが一致していること
- メッセージの不一致 → 直帰率の急上昇
- 商品広告 → 商品ページへ（総合カタログページではなく）

### ページスピード

| 読み込み時間 | 影響 |
|-------------|------|
| 1秒 → 3秒 | 直帰率 +32% |
| 1秒追加ごと | コンバージョン率 -4.42% |
| 3秒超過 | モバイル訪問者の53%が離脱 |
| 目標 | 3秒以内 |

#### スピード改善策

画像圧縮、JS最小化、CDN、遅延読み込み、AMP検討

### モバイルファースト（Xユーザーの80%以上がモバイル）

- レスポンシブデザインは必須
- 小さな画面でも読みやすいフォントサイズ
- 親指で操作しやすいボタンサイズ
- フォームフィールドは最小限（名前 + メールを出発点に）
- Flash/Silverlight/ActiveXは使用不可

### フォーム最適化

- 最小限のフィールド（名前 + メールをベースに）
- 可能な限りオートフィル対応
- X Lead Generation Cardsでプラットフォーム内完結

## 6. X Pixel実装ベストプラクティス

| ポイント | 詳細 |
|---------|------|
| ベースコード | すべてのページの `</head>` 前に配置 |
| ファーストパーティCookie | "Allow 1st party cookie" を常に有効にする |
| イベント配置 | コンバージョンタイプごとにサイトあたり1つのみ |
| URLマッチング | "Exact URL" ではなく "URL Contains" を使用 |
| 重複排除 | Pixel + CAPI併用時は `conversion_id` を設定 |
| iFrame不可 | Pixelコードをiframeや他のタグ内に配置しない |
| デバッグ | X Pixel Helper（Chrome拡張機能）を使用 |

## 7. クリエイティブテストフレームワーク

### X Ads Manager内蔵A/Bテスト（2023年1月〜）

- ベイジアンA/Bテスト（BayesAB）→ 「Win Chance」が出力指標
- 最大5広告グループ、各最大5広告
- **重要：A/Bテストのトグルはキャンペーン作成後に変更不可**
- DPAおよびTakeoverでは利用不可

### テスト優先順位

1. **クリエイティブアセット（画像/動画）** → ユーザーの97%がビジュアルに注目、最大のインパクト
2. **広告コピー** → 見出し、ベネフィットの切り口、文字数
3. **CTA** → ボタンテキスト、ソフト vs ハードCTA
4. **ターゲティング** → オーディエンスセグメント（クリエイティブ最適化の後に）

### テスト原則

- 1テストにつき1変数
- すべてのテストグループで同じオーディエンス・設定
- 最低2週間のテスト期間
- 変更後2〜3日は安定化を待つ

### テスト予算ガイド

| 企業規模 | テスト予算（広告費の%） |
|---------|----------------------|
| スタートアップ/小規模 | 10〜20% |
| 中規模（Xがメインチャネル） | 20〜40% |
| 大規模/ソーシャル重視 | 40〜60% |
| 初期推奨額 | $500〜$1,000/月 |

## 8. クリエイティブリフレッシュサイクル

### 推奨頻度

| 要素 | 推奨頻度 |
|------|---------|
| 基本的なリフレッシュ | 3〜4週間（パフォーマンスが落ちる前に先手で） |
| 高予算キャンペーン | 1〜2週間（$100K予算は1週間で消耗することがある） |
| ニッチオーディエンス | 2〜3週間（小規模オーディエンスはより早く疲弊） |
| 最低クリエイティブ数 | 5つ以上/広告グループ |

### クリエイティブ疲弊のサイン

| サイン | 意味 |
|--------|------|
| CTR低下 | ユーザーがクリックしなくなった |
| CPM/CPC上昇 | インプレッション効率が低下した |
| フリークエンシー上昇 + エンゲージメント低下 | 同じユーザーが繰り返し広告を見ているが反応していない |

## 9. フリークエンシーキャップ

- 広告想起の80%は週2回の露出で達成される
- 推奨：7日間で2〜3回をキャップに設定
- 広告グループレベルで設定（24時間/7日/30日単位）
- 事例：PepsiCoがフリークエンシーを3に制限 → コスト -18%、広告想起 +15%
