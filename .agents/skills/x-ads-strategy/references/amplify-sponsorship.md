# Amplify Sponsorship（アンプリファイ・スポンサーシップ）

## フォーマット概要

特定のパブリッシャーと1対1でペアリングされる動画広告。投稿レベルでのキャンペーン管理が可能。マネージドサービス専用（セルフサーブ非対応）であり、X営業チームとの直接契約が必要。

---

## 入稿スペック

| 項目 | 仕様 |
|------|------|
| ファイルサイズ | 最大1GB |
| 推奨尺 | 6秒以下を強く推奨 |
| 最大尺 | 2分20秒 |
| アスペクト比 | 16:9 または 1:1 |
| 推奨解像度 | 1200x1200 |

---

## 配信面・目的・課金

| 項目 | 詳細 |
|------|------|
| 配信面 | 特定パブリッシャーの動画コンテンツ再生前 |
| 広告目的 | Pre-roll Views（プレロール再生数）、Reach（リーチ） |
| 課金方式 | 個別交渉（X営業チームに問い合わせ） |

---

## ベストプラクティス

### クリエイティブ方針
- **6秒以下を強く推奨**：プレロールよりもさらに短尺が求められる
- **ペアリング先パブリッシャーとの親和性**がクリエイティブの最重要要素
- パブリッシャーのトーン＆スタイルに合わせた動画制作を行う
- ブランドの世界観とパブリッシャーのコンテンツが自然に融合する構成にする

### パブリッシャー選定基準
- ブランドのターゲット層とパブリッシャーの視聴者層の一致度
- コンテンツのトーンとブランドイメージの親和性
- パブリッシャーのエンゲージメント率と視聴者品質

---

## コピー構成

- スポンサーシップ専用のブランドメッセージを設計する
- パブリッシャーとのコラボレーションを表現する
- **簡潔に**：ブランド名＋ワンラインメッセージ

### コピーの基本構造
```
[ブランド名] × [パブリッシャー名/イベント名]
+ 一行のブランドメッセージ
```

### コピー例
```
「[ブランド名] presents [イベント名]」
「[ブランド名]がお届けする [体験/価値]」
```

---

## ファネル・フロー設計

```
Amplify Sponsorship（イベント期間中のシリーズ配信）
  → ブランド×パブリッシャーの認知獲得
    → Timeline Takeover との併用で最大効果
      → イベント後のリターゲティング
```

### ファネル設計のポイント
- **大型イベント（スポーツ大会、音楽フェスなど）に最適**
- 単発ではなく、イベント期間を通じたシリーズ配信が効果的
- **Sponsorship → Timeline Takeover の組み合わせ**で最大のインパクトを実現
- イベントの盛り上がりに合わせたクリエイティブのバリエーション展開を推奨
- イベント後はリーチしたユーザーに対してリターゲティング広告を配信する
