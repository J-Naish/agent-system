# X広告キャンペーン戦略リファレンス

---

## 1. キャンペーン構造

X広告は3層構造で管理される。

```
Campaign（キャンペーン）
├── 目的（Objective）の設定
├── 資金源（Funding source）の設定
│
├── Ad Group（広告グループ）※最大100グループ/キャンペーン
│   ├── 予算（Budget）
│   ├── 入札（Bidding）
│   ├── ターゲティング（Targeting）
│   ├── 配置（Placement）
│   │
│   ├── Ad（広告）※上限なし
│   │   └── クリエイティブ（Creative）
│   ├── Ad（広告）
│   │   └── クリエイティブ
│   └── ...
│
├── Ad Group（広告グループ）
│   └── ...
└── ...
```

**重要ルール:**

- キャンペーン・広告グループ・広告の3階層すべてがONでないと配信されない
- 同一広告グループ内の複数広告は、システムが自動的にパフォーマンスの良い広告を優先配信する
- CBO（Campaign Budget Optimization）を有効にすると、キャンペーン予算が広告グループ間で自動分配される

## 2. キャンペーン目的（9種類）

### 認知（Awareness）

| 目的 | 課金方式 | 概要 | ユースケース |
|------|----------|------|-------------|
| **Reach（リーチ）** | CPM | インプレッション最大化 | ブランド認知、商品ローンチ、イベント告知 |

### 検討（Consideration）

| 目的 | 課金方式 | 概要 | ユースケース |
|------|----------|------|-------------|
| **Video Views（動画再生）** | CPV | 動画再生数の最大化（2秒以上） | 商品デモ、ストーリーテリング |
| **Pre-roll Views（プレロール再生）** | CPV（6秒以上） | プレミアムパブリッシャーコンテンツ内再生。完視聴率が高い | ブランドリフト、高品質コンテンツ連動 |
| **App Installs（アプリインストール）** | CPM | アプリダウンロード促進。X経由のインストールは他媒体比+38% | アプリプロモーション |
| **Website Traffic（ウェブサイト訪問）** | CPC | ウェブサイトへの訪問促進。コスト効率が良い | サイト誘導、記事閲覧 |
| **Engagements（エンゲージメント）** | CPE | いいね・リポスト・リプライの最大化 | コミュニティ構築、UGC促進 |
| **Keywords（キーワード）** | CPM | 特定の検索キーワードをターゲティング。2025年新機能。購買意欲の高いユーザーにリーチ | 検索連動型広告 |

### コンバージョン（Conversion）

| 目的 | 課金方式 | 概要 | ユースケース |
|------|----------|------|-------------|
| **Sales（セールス）** | CPM | 購入・サインアップ促進。2025年新機能。AI最適化に4〜6週間必要 | EC販売、リード獲得 |
| **App Re-engagement（アプリ再エンゲージメント）** | CPC | 休眠ユーザーの再活性化 | 新機能告知、特別オファー |

**注意:** フォロワー目的（Promoted Accounts）は2023年に廃止済み。フォロワー獲得にはReachまたはEngagement目的を代用する。

## 3. ターゲティング

### A. デモグラフィック（5種類）

| 種類 | 詳細 |
|------|------|
| **ロケーション** | 国、地域、都市圏、市区町村、郵便番号、半径指定（1〜50マイル、米国・日本のみ） |
| **言語** | 投稿・エンゲージメントパターンから判定。注意：日本語設定するとオーディエンスが約20%減少する場合がある |
| **デバイス/プラットフォーム/Wi-Fi** | iOS、Android、PC、OSバージョン、キャリア、新規デバイス、Wi-Fi接続 |
| **年齢** | 13-17、18-24、25-34、35-49、50-54、55-64、65+ |
| **性別** | 男性、女性、すべて |

### B. 行動ベース（6種類）

| 種類 | 詳細 |
|------|------|
| **会話ターゲティング** | 25以上のカテゴリ、10,000以上のトピック |
| **イベントターゲティング** | 主要なオンライン・オフラインイベント（開催2週間前〜3週間後） |
| **キーワードターゲティング** | 検索語句、投稿内容、キーワード・ハッシュタグへのエンゲージメント。除外キーワード設定可。広告グループあたり最大750件、25件以上推奨 |
| **映画/TV番組ターゲティング** | 特定の映画・TV番組について投稿しているユーザー |
| **興味関心ターゲティング** | 25カテゴリ、350以上のプリセット興味関心 |
| **フォロワー類似ターゲティング** | 特定アカウントのフォロワーに類似したユーザーをターゲット |

### C. カスタムオーディエンス（3種類）

| 種類 | 詳細 |
|------|------|
| **フォロワーターゲティング** | 自アカウントのフォロワー |
| **カスタムオーディエンス** | リストアップロード（メール、電話番号）、ウェブサイト訪問者（Pixel）、アプリ活動。最低100ユーザー必要 |
| **投稿エンゲージャーリマーケティング** | 過去のキャンペーンやオーガニック投稿にエンゲージしたユーザー |

### ターゲティングロジック

- **加算型（OR）:** 興味関心、キーワード → いずれかに一致すれば対象
- **制限型（AND）:** デモグラフィック（年齢、性別、地域） → すべての条件を満たす必要あり

### その他の機能

- **Audience Expansion（オーディエンス拡張）:** 設定ターゲットに類似するユーザーへ自動拡張
- **Optimization Targeting（最適化ターゲティング）:** AI駆動のターゲティング自動最適化
- **Do Not Reach リスト:** 除外リスト
- **Behavioral Targeting（行動ターゲティング）:** パートナーデータ利用（米国・英国のみ）
- **Catalog Activity Audiences:** DPA（動的商品広告）用のカタログ行動オーディエンス

### 推奨ターゲティング戦略

1. **AI駆動型:** デモグラフィックのみ設定 → X AIが最適なユーザーを自動発見
2. **手動型:** キーワード30件以上 + フォロワー類似アカウント10件以上を設定し、幅広くリーチ

## 4. 入札・予算・課金

### オークション方式

**セカンドプライスオークション:** 2番目に高い入札者より$0.01高い金額を支払う

```
広告スコア = 入札金額 + 品質スコア（共鳴度 + 関連性 + 新鮮度）
```

### 入札戦略

| 戦略 | 概要 | 推奨 |
|------|------|------|
| **Auto bid（自動入札）** | 最安値で自動最適化 | 初心者向け推奨 |
| **Max bid（上限入札）** | CPV/CPCの上限を設定 | コスト管理重視 |
| **Target bid（目標入札）** | 目標コストに向けて最適化 | 一部の目的で利用可 |
| **Target CPA（目標CPA）** | コンバージョン単価の目標に向けて最適化。Sales目的、2025年新機能 | コンバージョン重視 |

### 予算タイプ

- **日額予算（Daily budget）:** 必須
- **通算予算（Total/Lifetime budget）:** 任意
- **広告グループ予算（Ad group budget）:** 任意
- **キャンペーン支出上限（Campaign spend cap）:** 任意

### 配信ペース

- **Standard（標準）:** 均等配信
- **Accelerated（加速）:** 高速消化

## 5. 計測・アトリビューション

### X Pixel

JavaScriptタグ方式（ベースコード + イベントコード）。

- **ベースコード:** 全ページに設置
- **イベントコード:** 特定ページに設置
- **対応イベント:** 14種類（Page View、Purchase、Add to Cart など）
- **Cookie:** ファーストパーティCookie（Click ID）
- **プライバシー:** PII（個人識別情報）は自動SHA256ハッシュ化

### Conversions API（CAPI）

サーバー間通信方式（ウェブサイトコード不要）。

- **要件:** Developer Account + 最低1つの識別子（twclid、メール、電話番号）
- **対応パートナー:** Adobe、Tealium、Metarouter、Datahash、Rudderstack
- **重複排除:** Pixel + CAPI併用時はデデュプリケーションキーが必要

### アトリビューションウィンドウ

| 種類 | 設定範囲 | デフォルト |
|------|---------|-----------|
| ポストエンゲージメント | 1〜90日 | 14日 |
| ポストビュー | オフ〜30日 | 1日 |

**注意:** データ確定には24〜48時間かかる

## 8. 戦略設計ガイド

### Phase 1: 目的選定（ファネルポジション別）

| ファネル | 推奨目的 |
|---------|---------|
| 認知（Awareness） | Reach |
| 検討（Consideration） | Video Views、Website Traffic、Engagements、Keywords |
| コンバージョン（Conversion） | Sales、App Installs、App Re-engagement |

### Phase 2: フォーマット選定マトリクス

ビジネス目標に応じて最適なフォーマットを選定する。

### Phase 3: ターゲティング戦略（温度別）

| オーディエンス温度 | 戦略 |
|-------------------|------|
| **Cold（新規）** | 興味関心・キーワード・フォロワー類似で幅広くリーチ |
| **Warm（中間）** | サイト訪問者・動画視聴者・エンゲージャーをリマーケティング |
| **Hot（見込み客）** | カート放棄者・リスト顧客をピンポイントでターゲット |

### Phase 4: 予算配分

| フェーズ | 期間 | 予算目安 |
|---------|------|---------|
| テスト期間 | 1〜2ヶ月 | ¥100,000以上 |
| スケール期間 | 3ヶ月以上 | ¥200,000以上 |

**ファネル別予算配分比率（認知/検討/コンバージョン）:**

| ステージ | 認知 | 検討 | コンバージョン |
|---------|------|------|--------------|
| 初期 | 50% | 30% | 20% |
| 成長期 | 30% | 30% | 40% |
| 成熟期 | 20% | 30% | 50% |

### Phase 5: 最適化サイクル

| タイミング | アクション |
|-----------|----------|
| 1週間 | ターゲティングチェック |
| 2〜3週間 | クリエイティブA/Bテスト結果確認 |
| 4〜6週間 | AI安定化期間 |
| 継続 | クリエイティブ疲労モニタリング |

### ラーニングフェーズについて

- tCPA（目標CPA）は約10コンバージョン後にアクティベート
- ラーニング期間は3〜5日
- ラーニング中は設定変更を避ける
- Sales目的は4〜6週間のデータ蓄積が必要

## 9. 命名規則

### キャンペーン命名

```
[Platform]_[Objective]_[Target]_[Format]_[Date]
```

**例:** `X_Sales_JP-25to34_Video_202602`

### 広告グループ命名

```
AG_[Strategy]_[Audience]_[Detail]
```

**例:** `AG_Retargeting_SiteVisitors_7d`

### 広告命名

```
Ad_[Product]_[Format]_[Version]
```

**例:** `Ad_ProductA_15sVideo_v1`

### UTMパラメータ

| パラメータ | 値 |
|-----------|-----|
| utm_source | `x` |
| utm_medium | `paid_social` |
| utm_campaign | `{campaign_name}` |
| utm_content | `{ad_name}` |
| utm_term | `{keyword}` |
