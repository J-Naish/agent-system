# X広告フォーマット：画像広告

---

## フォーマット概要

画像広告はX広告で最も一般的なフォーマットであり、1枚の画像とテキストを組み合わせて商品・サービスを訴求する。タイムライン上で視覚的なインパクトを与え、テキスト広告よりも高いエンゲージメント率が期待できる。

画像広告にはスタンドアロン型に加え、ウェブサイトカード、アプリカード、カンバセーションボタン、投票（ポール）などのサブフォーマットが存在し、目的に応じて使い分ける。

## クリエイティブベストプラクティス

### 画像デザインルール

- **テキストオーバーレイは画像面積の20〜25%以内**：テキストが多すぎるとバナー広告のように見え、エンゲージメントが低下する
- **セーフゾーン：画像の端から10%のマージン**を確保する。UIオーバーレイや切り取りで重要な情報が隠れるのを防ぐ
- **ロゴは左上に配置**：視線の流れ（Zパターン）に沿い、ブランド認知を早期に確保する
- **モバイルは1:1（正方形）を推奨**：モバイル画面での表示面積が最大化される
- **ライフスタイル画像 > 製品のみの画像**：使用シーンを見せるライフスタイル画像は、製品単体画像より約20%エンゲージメントが向上する
- **高コントラストの配色**：タイムラインのスクロール中に目を引くため、明暗のコントラストを強くする
- **人の顔を入れる**：顔が含まれる画像はエンゲージメント率が向上する傾向にある
- **フォーカルポイントは1つ**：複数の要素に視線が分散しないよう、1つの明確な主役を設定する
- **鮮やかで明るい色彩**：タイムラインで埋もれないよう、ビビッドな色使いを心がける

### よくある間違い

| 間違い | 問題点 |
|--------|--------|
| テキストが多すぎる | バナー広告のように見え、スキップされる |
| 低解像度の画像 | ブランドの信頼性が低下する |
| 1枚に複数商品を詰め込む | フォーカルポイントが分散し、何の広告かわからなくなる |
| 白背景のみ | タイムラインの白背景と同化し、広告が目立たない |
| ストック写真感が強い | ユーザーに広告と即座に判断され、スキップされる |

## 広告コピー構成

### 基本コピーフレームワーク

```
[フック（1行目）]  ← スクロールを止める一言
  ↓
[ベネフィット/価値の提示]  ← なぜユーザーに関係があるか
  ↓
CTA  ← ウェブサイトカードボタンで行動を促す
```

### 画像とコピーの関係

- **画像がメインメッセージを担う**：コピーは補完的な役割
- **画像とコピーで同じ情報を繰り返さない**：画像で伝えていることをテキストでも書くのは冗長
- 画像が「何を」伝え、コピーが「なぜ」を補足するのが理想的な分担

### サブフォーマット別コピー設計

| サブフォーマット | コピーの役割 |
|----------------|-------------|
| スタンドアロン | ブランドメッセージを伝える。画像+テキストで完結する設計 |
| ウェブサイトカード | ヘッドラインでLPの内容をプレビュー（50文字以内推奨）。クリック後の期待値を設定する |
| アプリカード | 1つの具体的なベネフィットに絞る。機能ではなくユーザーが得る価値を訴求 |
| 投票（ポール） | 選択肢自体がコピーの役割を果たす。投票したくなる設問設計が重要 |
| カンバセーションボタン | ユーザーが自発的に投稿したくなるハッシュタグを設計する |

## ファネル/フロー設計

### 基本フロー

```
広告（画像 + コピー + カードヘッドライン）
  ↓
クリック
  ↓
LP（広告と同じビジュアル・メッセージ）
  ↓
コンバージョン
```

### メッセージマッチの原則

広告からLPまでの一貫性が最も重要な成功要因である。

- **メッセージマッチは必須**：広告のコピー・ビジュアルとLPの内容が一致していないと離脱率が急増する
- **広告と同じ画像をLPにも使用**：同じ画像がLP上にも表示されることで、ユーザーは正しいページに来たと認識し、直帰率が低下する
- **モバイルファースト設計**：X広告ユーザーの80%以上がモバイルからアクセスする。LP は必ずモバイル最適化する
- **LP読み込み速度は3秒以内**：3秒を超えると離脱率が大幅に上昇する

### サブフォーマット別ファネル設計

#### スタンドアロン画像広告

```
画像広告（ブランドメッセージ）
  ↓
エンゲージメント（いいね、RT、リプライ）
  ↓
ブランド認知の蓄積
  ↓
リターゲティング広告（ウェブサイトカード付き）
  ↓
LP → コンバージョン
```

#### ウェブサイトカード

```
画像広告 + ウェブサイトカード
  ↓
ヘッドラインでLP内容をプレビュー
  ↓
クリック → LP
  ↓
広告と同じビジュアル・メッセージで迎える
  ↓
コンバージョン
```

#### アプリカード

```
画像広告 + アプリカード
  ↓
1つのベネフィットを訴求
  ↓
CTAボタン（Install/Open/Play等）
  ↓
App Store / Google Play
  ↓
インストール → アプリ内コンバージョン
```

#### 投票（ポール）付き

```
画像広告 + 投票
  ↓
ユーザーが投票に参加
  ↓
エンゲージメント + ブランド想起
  ↓
投票結果をフォローアップコンテンツとして活用
  ↓
リターゲティング → コンバージョン
```

#### カンバセーションボタン付き

```
画像広告 + カンバセーションボタン
  ↓
ユーザーがハッシュタグ付き投稿を作成
  ↓
UGC（ユーザー生成コンテンツ）の拡散
  ↓
オーガニックリーチの拡大
  ↓
ブランド認知 → 検索 → コンバージョン
```

## まとめ

画像広告はX広告の主力フォーマットであり、ビジュアルの質とメッセージマッチがパフォーマンスを左右する。画像がメインメッセージを担い、コピーが補完するという役割分担を明確にし、広告からLPまで一貫したビジュアル・メッセージ体験を設計することが成功の鍵である。モバイルファーストの設計思想を徹底し、LP読み込み速度3秒以内を必ず守ること。
