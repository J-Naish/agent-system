# Branded Notifications（ブランド通知）

## フォーマット概要

Branded Notificationsは、ユーザーが自らオプトインすることで、ブランドからのスケジュール配信投稿を自動的に@メンション付きで受け取れる広告フォーマット。ユーザーの能動的な同意に基づくため、高いエンゲージメント率と到達率を実現する。

### 特徴

- オプトイン方式: ユーザーが「いいね」やリポストでオプトインし、スケジュールされたタイミングでブランドからの@メンション投稿を自動受信する
- 3つのキャンペーンタイプに対応
- ユーザー主導の同意モデルにより高い開封率・エンゲージメント率

### 利用要件

- X認証済みアカウント（Verified）
- Arrow（IC Group）OAuthアクセス権限
- 公開アカウントのみ対応（非公開アカウント不可）
- オプトアウト機構の実装が必須

## スペック・入稿規定

### キャンペーンタイプ別仕様

| タイプ | CTA〜通知間隔 | オプトイン回数 | 通知回数 | 用途 |
|--------|---------------|---------------|---------|------|
| Schedule Notification | 最大30日 | 1回 | 1回（スケジュール指定） | 単発イベント告知、製品発売日通知 |
| Subscription Notification | 各通知30日以内 | 1回 | 複数回（定期配信） | 連載コンテンツ、定期キャンペーン |
| Instant Notification | CTA後30日以内 | 1回 | 1回（オプトイン直後に即時配信） | 即時特典配布、限定情報の即時提供 |

### 共通要件

| 項目 | 仕様 |
|------|------|
| アカウント要件 | X認証済み + Arrow OAuth |
| アカウント公開設定 | 公開アカウントのみ |
| オプトアウト | 必須（ユーザーがいつでも解除可能な仕組み） |
| オプトイン方法 | いいね / リポスト |
| 通知方法 | @メンション付き投稿として配信 |

## ベストプラクティス

### オプトイン投稿の設計

- オプトインによって「何が届くか」「なぜ登録すべきか」を明確に伝える
- オプトインのハードルを最小化する（いいねだけでOKなど）
- ビジュアル（画像/動画）で期待感を高め、テキストだけに頼らない
- オプトアウト方法を明示する（義務要件）

### 通知投稿の設計

- オプトイン時の約束を確実に履行する内容にする
- 明確なCTA（LP遷移、購入、詳細確認など）を含める
- 通知タイミングはユーザーのアクティブ時間帯に合わせる
- 連載型（Subscription）の場合、次回予告を含めて継続期待を醸成する

### キャンペーンタイプ別の活用シーン

| ユースケース | 推奨タイプ |
|-------------|-----------|
| 製品ローンチのカウントダウン | Schedule Notification |
| イベント当日リマインダー | Schedule Notification |
| 定期コンテンツシリーズ配信 | Subscription Notification |
| 限定プロモーション告知 | Instant Notification |

### 成功事例

- **Showtime / Shameless キャンペーン**: ツイート量 +276%、会話ボリューム +65% を達成

## コピー構成

### オプトイン投稿（CTA投稿）

```
[何が届くかを明確に伝える]
[なぜ登録すべきかの価値提示]

[オプトイン方法の説明]

例:
「○月○日の新製品発表、いち早くお届けします。
この投稿にいいねで、発表当日に通知が届きます。」
```

**ポイント**:
- 「何が届くか」+「なぜ登録すべきか」の2要素を必ず含める
- オプトイン方法を具体的に記載する（「いいねで登録」など）
- オプトアウト機構の表示は必須

### 通知投稿（配信投稿）

```
[約束した内容の提供]
[明確なCTA（LP / 購入 / 詳細）]

例:
「お待たせしました！新製品○○が本日発売。
先行予約いただいた方限定の特典はこちら👇
[LP URL]」
```

**ポイント**:
- オプトイン時の約束を確実に果たす内容にする
- 具体的な次のアクション（CTA）を明記する
- LP、購入ページ、キャンペーンサイトへの動線を設置する

### タイプ別コピーの方向性

| タイプ | コピーの方向性 | 表現例 |
|--------|--------------|--------|
| Schedule | カウントダウン型 | 「あと○日」ビジュアルシリーズ |
| Subscription | 連載予告型 | 「毎週○曜日に配信」 |
| Instant | 即時性訴求型 | 「今すぐオプトインで限定情報」 |

## ファネル・フロー設計

### 基本フロー

```
プロモーテッド投稿（オプトインCTA）
    ↓
ユーザーが「いいね」/「リポスト」でオプトイン
    ↓
スケジュールされたタイミングで通知配信（@メンション）
    ↓
通知投稿（CTA付き）
    ↓
LP / キャンペーンサイト / 購入ページ
```

### 活用パターン別フロー

#### パターン1: 製品ローンチ（Schedule Notification）

```
[2週間前] ティーザー広告でオプトイン獲得
    ↓
[1週間前] カウントダウン投稿シリーズ（あと7日→3日→1日）
    ↓
[発売当日] Schedule Notification配信（@メンション）
    ↓
通知投稿: 製品詳細 + 購入CTA
    ↓
LP / ECサイト
```

#### パターン2: イベントリマインダー（Schedule Notification + X Live連携）

```
[1週間前] オプトイン促進広告
    ↓
[前日] リマインダー通知配信
    ↓
[当日] X Live配信告知通知
    ↓
X Liveイベントページ → ライブ視聴
```

#### パターン3: 定期コンテンツ配信（Subscription Notification）

```
[初回] オプトイン投稿「毎週○曜日に○○をお届け」
    ↓
[毎週] 定期通知配信（連載コンテンツ / 限定情報）
    ↓
各通知投稿にCTA（LP / 記事 / 商品ページ）
    ↓
継続接触によるブランド親和性向上 → コンバージョン
```

#### パターン4: 即時特典配布（Instant Notification）

```
プロモーテッド投稿「いいねで限定クーポンを即時配信」
    ↓
ユーザーがいいねでオプトイン
    ↓
即時通知（@メンション）: クーポンコード + 購入CTA
    ↓
LP / ECサイトでクーポン適用 → 購入
```

### 最適な活用タイミング

- **プリローンチ期の期待醸成**: 発売前のカウントダウンで話題を作り、発売日に一斉通知
- **イベントリマインダー**: X Liveやオフラインイベントの視聴者/参加者を事前に確保
- **定期コンテンツのファン化**: Subscription型で継続接触し、ブランドロイヤルティを醸成
- **即時転換施策**: Instant型でオプトインから購入までの導線を最短化

### 運用上の注意点

- オプトアウト機構は必ず実装し、通知投稿内にも解除方法を表示する
- CTA投稿から通知配信まで最大30日の制約を考慮してスケジュールを設計する
- Subscription型は各通知間隔が30日以内である必要があるため、配信カレンダーを事前に作成する
- Arrow（IC Group）のOAuth連携設定は、キャンペーン開始前に余裕をもって完了させる
- オプトイン数の推移を日次でモニタリングし、プロモーション投稿の追加出稿を判断する
