# X Live（ライブ配信広告）

## フォーマット概要

X Liveは、リアルタイムのライブ動画配信をプロモーションとして既存フォロワー以外にも拡散できる広告フォーマット。イベント、製品発表、Q&Aセッションなどをライブで届け、視聴者とのリアルタイムインタラクションを通じてブランドエンゲージメントを最大化する。

### 特徴

- リアルタイムライブ動画をプロモーション配信
- 既存フォロワー以外のユーザーにもリーチ可能
- イベントページによる事前集客と情報集約
- Timeline Takeoverとの組み合わせ配信が可能

### 配信面

- タイムライン
- イベントページ
- Timeline Takeoverとの併用可能

### 広告目的

- リーチ（Reach）
- 動画再生数（Video Views）
- エンゲージメント（Engagements）

---

## スペック・入稿規定

### ストリーミング仕様（RTMP）

| 項目 | 推奨値 | 最大値 |
|------|--------|--------|
| 映像コーデック | H.264 / AVC | - |
| ビットレート | 9 Mbps | 12 Mbps |
| 音声コーデック | AAC-LC | - |
| 音声ビットレート | - | 128 kbps |
| 解像度 | 1280 x 720 | 1920 x 1080 |
| フレームレート | 30 FPS | 60 FPS |
| キーフレーム間隔（OBS） | 3秒ごと | - |

### イベントページ仕様

| 項目 | 仕様 |
|------|------|
| 説明文 | 最大280文字（外部リンク挿入可） |
| カルーセル | 最大5本（ライブ / VOD / GIF / 写真） |
| タイムライン投稿 | 最大5キーワード |

## ベストプラクティス

### 配信前準備（最低1週間前から開始）

- **ティーザー広告**: ライブ配信の予告動画/画像をプロモーション配信し、認知を拡大する
- **Branded Notification**: リマインダー通知のオプトインを促進し、配信当日の視聴者を事前に確保する
- **イベントページ作成**: 説明文280文字以内で配信内容を明記し、カルーセルに最大5つのメディアを設定する
- **配信テスト**: OBS設定の確認、キーフレーム間隔3秒、安定したインターネット接続のテストを事前に実施する

### 配信中

- **推奨設定**: 1280x720、9Mbps、30FPSで安定配信を優先する
- **リアルタイムインタラクション**: 投票（ポール）、Q&A、ライブコメントを活用して視聴者参加を促す
- **ネットワーク**: 有線接続を推奨。Wi-Fiの場合は5GHz帯を使用し、他デバイスの帯域使用を制限する
- **進行**: 開始5分以内に本題に入り、15〜20分間隔でインタラクションポイントを設ける

### 配信後

- **ハイライトクリップ作成**: 配信のベストモーメントを30秒〜1分のクリップに編集する
- **Amplify Pre-roll素材化**: ハイライトクリップをAmplify Pre-rollコンテンツとして二次活用する
- **通常動画広告として再配信**: 編集済みクリップを通常の動画広告として継続的にプロモーションする
- **視聴者リターゲティング**: ライブ視聴者をカスタムオーディエンスとして蓄積し、後続キャンペーンに活用する

## コピー構成

### 配信前ティーザー投稿

```
○月○日 ○時〜 ライブ配信

[配信内容のプレビュー / 見どころ]

[イベントページURL or オプトインCTA]
```

**ポイント**:
- 日時を冒頭に明記して「いつ」を即座に伝える
- 配信で何が得られるかを具体的に記載する
- Branded Notificationへのオプトイン誘導を含める

### 配信中エンゲージメント投稿

```
[質問 / 投票 / 意見募集]

例:
「どちらが気になりますか？」
「質問があればコメントで教えてください」
「○○について知りたいことは？」
```

**ポイント**:
- 視聴者が即座に反応できるシンプルな質問にする
- ポールやQ&Aを活用して参加のハードルを下げる
- 視聴者のコメントをリアルタイムで取り上げて一体感を演出する

### 配信後ハイライト投稿

```
[ライブ配信のまとめ / ハイライト]

[主要な発表内容 / ベストモーメント]

見逃した方はこちら👇
[アーカイブリンク]
```

**ポイント**:
- 配信の要点を簡潔にまとめる
- アーカイブへの動線を明確に設置する
- ハイライトクリップと組み合わせて投稿する

## ファネル・フロー設計

### 全体フロー

```
[1週間前] ティーザー広告 + Branded Notification オプトイン促進
    ↓
[配信当日] X Live配信 + Timeline Takeover コンボ
    ↓
[配信直後] ハイライトクリップ作成・配信
    ↓
[配信後1〜2週間] 通常動画広告として再配信
    ↓
[継続] 視聴者リターゲティング → コンバージョン広告
```

### フェーズ別詳細

#### Phase 1: 事前集客（配信1週間前〜前日）

| 施策 | 目的 | KPI |
|------|------|-----|
| ティーザー動画/画像広告 | 配信の認知拡大 | インプレッション、リーチ |
| Branded Notification | リマインダーオプトイン獲得 | オプトイン数 |
| イベントページ公開 | 詳細情報の集約 | ページ閲覧数 |

#### Phase 2: 配信当日

| 施策 | 目的 | KPI |
|------|------|-----|
| X Live配信 | リアルタイムエンゲージメント | 同時視聴者数、視聴時間 |
| Timeline Takeover併用 | 配信のリーチ最大化 | インプレッション |
| リアルタイムインタラクション | 視聴者参加促進 | コメント数、ポール回答数 |

#### Phase 3: 配信後活用（配信後〜2週間）

| 施策 | 目的 | KPI |
|------|------|-----|
| ハイライトクリップ配信 | 見逃し層への訴求 | 動画再生数 |
| Amplify Pre-roll素材化 | プレミアムコンテンツでの露出 | 完視聴率 |
| 通常動画広告化 | 継続的なリーチ拡大 | リーチ、エンゲージメント |

#### Phase 4: コンバージョン（配信後2週間〜）

| 施策 | 目的 | KPI |
|------|------|-----|
| 視聴者リターゲティング | 興味関心層への再アプローチ | CTR |
| コンバージョン広告 | 購入・申込などの成果獲得 | CVR、CPA |

### 運用上の注意点

- 配信前の準備期間は最低1週間を確保する
- ティーザー広告は配信日の5〜7日前から開始し、配信前日にリマインド投稿を行う
- 配信中はモデレーターを配置し、コメント管理とインタラクション対応を分担する
- ハイライトクリップは配信後24時間以内に公開し、鮮度を維持する
- Amplify Pre-rollへの素材転用は、クリップの完視聴率を確認してから判断する
