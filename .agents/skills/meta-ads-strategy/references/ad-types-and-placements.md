## 1. キャンペーン目的（Campaign Objectives）

Meta広告は6つのキャンペーン目的に統合されている。目的の選択はアルゴリズムの最適化方向を決定するため、ビジネスゴールとの正確な一致が最重要。

> **重要:** 目的を誤ると最適化の方向がずれる。例えば「購入」が目的なのに「Traffic」を選ぶと、クリックしやすいユーザーに配信され、購入しやすいユーザーには届かない。

### 1-1. 各目的の詳細

#### Awareness（認知）

| 項目 | 内容 |
|------|------|
| 概要 | ブランド認知・リーチを最大化。ファネルの最上部 |
| パフォーマンス目標 | リーチ最大化、インプレッション、広告想起リフト |
| 使うタイミング | 新規ブランド立ち上げ、新市場参入、イベント告知、新商品ローンチ |
| 適したビジネス | 全業種（立ち上げ期）、ブランド認知が低い企業 |
| 推奨フォーマット | 動画/リールズ（最も効果的）、画像、カルーセル |
| 備考 | ToFキャンペーンとして、リターゲティング用オーディエンス蓄積にも活用 |

#### Traffic（トラフィック）

| 項目 | 内容 |
|------|------|
| 概要 | 特定URLへのクリックを最大化 |
| パフォーマンス目標 | ランディングページビュー、リンククリック、デイリーユニークリーチ |
| 使うタイミング | ブログ記事の拡散、ECサイトへの送客、LP送客、コンテンツマーケティング |
| 適したビジネス | メディア/ブログ、アフィリエイト、メール収集LP、Pixelデータが不足している新規アカウント |
| 推奨フォーマット | 画像、カルーセル、動画 |
| 備考 | CVデータが不足している初期段階で使用し、データ蓄積後にSalesへ移行 |

#### Engagement（エンゲージメント）

| 項目 | 内容 |
|------|------|
| 概要 | いいね・シェア・コメント・動画視聴・メッセージ等のインタラクションを促進 |
| パフォーマンス目標 | 投稿エンゲージメント、動画視聴、メッセージ、イベント参加 |
| 使うタイミング | ソーシャルプルーフの蓄積、コミュニティ構築、イベント集客、動画視聴促進 |
| 適したビジネス | コンテンツクリエイター、イベント運営、コミュニティ重視のブランド |
| 推奨フォーマット | 動画/リールズ、カルーセル |
| 備考 | 旧「メッセージ」「コンバージョン」「動画再生」目的を統合 |

#### Leads（リード獲得）

| 項目 | 内容 |
|------|------|
| 概要 | 見込み客の連絡先情報を取得 |
| パフォーマンス目標 | インスタントフォーム送信、ウェブサイトCV、Messenger、電話 |
| 使うタイミング | メルマガ登録、ウェビナー登録、資料DL、問い合わせ獲得、デモ依頼 |
| 適したビジネス | B2Bサービス、不動産、教育、コンサルティング、士業、保険 |
| 推奨フォーマット | リード広告（Instant Form）、動画、画像 |
| 2つの方式 | **Instant Form（プラットフォーム内）:** 量が多いがリード品質は低め / **ウェブサイトCV:** 量は少ないがエンゲージメントと品質が高い |

#### App Promotion（アプリ促進）

| 項目 | 内容 |
|------|------|
| 概要 | アプリインストール・アプリ内アクションを促進 |
| パフォーマンス目標 | アプリインストール、アプリイベント、バリュー（ROAS最大化） |
| 使うタイミング | アプリインストール促進、既存ユーザーのエンゲージメント向上 |
| 適したビジネス | モバイルアプリ、ゲーム |
| 推奨フォーマット | 動画（アプリUXを動画でデモ）、プレイアブル広告（インストール率2〜3倍向上） |
| 備考 | 50件以上のインストール蓄積後に高価値イベントで最適化 |

#### Sales（売上）

| 項目 | 内容 |
|------|------|
| 概要 | 購入・申込等の直接的なコンバージョンを最大化 |
| パフォーマンス目標 | 購入、カートに追加、決済開始、ROAS最大化 |
| 使うタイミング | 商品購入、カタログ販売、プロモーション販売、リターゲティング |
| 適したビジネス | EC全般、D2C、サブスクリプション、高額サービスのCV獲得 |
| 推奨フォーマット | コレクション、カルーセル、動的商品広告、動画 |
| 前提条件 | Pixel/CAPIが正しく設置済み、CVデータの蓄積がある状態が理想 |
| 備考 | **ECやリード獲得が主目的なら、最初からSalesを選ぶのがデフォルト** |

### 1-2. ビジネスタイプ別の目的選定

| ビジネスタイプ | 推奨キャンペーン目的 | 最適化イベント | 備考 |
|---------------|--------------------|--------------|----- |
| **EC / D2C** | Sales（ASC推奨） | Purchase | CVデータ不足時はAdd to Cart で開始 |
| **リード獲得（B2B）** | Leads または Sales | Lead / Form Submit | Instant Form vs ウェブサイトCVを比較テスト |
| **SaaS** | Leads → Sales | Demo Request / Sign Up | デモ依頼をCV地点に設定 |
| **店舗集客** | Awareness + Sales | Store Visit / Purchase | 地域ターゲティングと組み合わせ |
| **アプリ** | App Promotion | Install → In-App Event | インストール後の高価値イベントに段階的に移行 |
| **メディア/ブログ** | Traffic | Landing Page View | 広告収入・メール収集が目的 |
| **イベント** | Awareness + Engagement | Event Response / Reach | イベント前はAwareness、直前はEngagement |
| **ブランド構築** | Awareness | Reach / Ad Recall Lift | 動画中心のブランドストーリーテリング |

### 1-3. 最適化イベントの選び方

最適化イベントは「最も価値のあるアクション」を選ぶ。ただし、Metaのアルゴリズムは週50件以上の最適化イベントを必要とする。

```
理想: Purchase（購入）で最適化
  ↓ 週50件未満の場合
次善: Add to Cart / Initiate Checkout
  ↓ それでも不足の場合
代替: View Content / Landing Page View
  ↓ CVデータ蓄積後
Purchase に最適化を切り替え
```

**注意:** 上位ファネルイベント（View Content等）で最適化すると、量は増えるが質（購入意向）は下がる。可能な限り下位ファネルイベントを選択する。

## 2. 広告フォーマット（Ad Formats）

### 2-1. 画像広告（Image Ads）

**概要:** 1枚の静止画＋テキスト＋CTAボタン。最も基本的なフォーマット。

| 項目 | 仕様 |
|------|------|
| 推奨解像度 | 1080×1080px（1:1）、1080×1350px（4:5）、1080×1920px（9:16） |
| 最小解像度 | 600×600px |
| Retina対応 | 1440×1440px以上推奨 |
| ファイル形式 | JPG、PNG |
| 最大ファイルサイズ | 30MB |

**パフォーマンス特性:**
- 全体CVの約 **60〜70%** は依然として静止画広告から発生
- 4:5フォーマットは1:1より最大 **15%** パフォーマンス向上
- 制作が速く、テストサイクルを高速に回せる

**選ぶべきケース:** 単一商品やオファーの訴求、制作リソースが限られている場合、迅速なA/Bテスト、リターゲティング

### 2-2. 動画広告（Video Ads）

**概要:** 動画コンテンツを活用。2026年のMeta広告在庫の90%は縦型動画。

| 項目 | 仕様 |
|------|------|
| 推奨解像度 | 1080×1080px（1:1）、1080×1350px（4:5）、1080×1920px（9:16） |
| 推奨長さ | フィード: 15〜60秒、リールズ: 15〜30秒、ストーリーズ: 最大15秒 |
| ファイル形式 | MP4、MOV |
| 最大ファイルサイズ | 4GB |
| コーデック | H.264、AAC音声128kbps以上 |

**パフォーマンス特性:**
- リールズ広告は他の配置面より平均 **13%** 高いROAS、**16%** 低いCPA
- ユーザーの80%は音声OFFで視聴 → **字幕は必須**

**選ぶべきケース:** 商品デモ、ブランドストーリーテリング、テスティモニアル、感情に訴えるメッセージ

### 2-3. カルーセル広告（Carousel Ads）

**概要:** 2〜10枚の画像/動画をスワイプで閲覧可能。各カードに個別のリンク設定が可能。

| 項目 | 仕様 |
|------|------|
| カード数 | 2〜10枚（推奨: 3〜5枚から開始） |
| 推奨解像度 | 1080×1080px（1:1） |
| アスペクト比 | 全カード統一（1:1 または 4:5） |
| 動画カード | 15秒以内推奨 |

**パフォーマンス特性:**
- 単一画像リンク広告と比較して **CPA 30〜50%低下**、**CPC 20〜30%低下**

**選ぶべきケース:** 複数商品の紹介、ステップバイステップの説明、ビフォーアフター、ベネフィットの積み上げ

### 2-4. コレクション広告（Collection Ads）

**概要:** カバー画像/動画＋商品グリッドのモバイル特化フォーマット。タップするとInstant Experienceが展開。

| 項目 | 仕様 |
|------|------|
| カバー | 画像または動画1つ |
| 商品画像 | 1:1（正方形）で自動表示 |
| 前提条件 | Meta Business Manager に商品カタログ接続が必要 |

**パフォーマンス特性:** EC向けフォーマットで最も高いCVR傾向。購入までの摩擦を大幅に削減。

**選ぶべきケース:** 商品カタログが充実しているEC、複数商品をブラウジングさせたい場合

### 2-5. リード広告（Lead Ads / Instant Forms）

**概要:** Facebook/Instagram内でフォーム入力が完結するフォーマット。

**パフォーマンス特性:** プラットフォーム外への遷移が不要で離脱率低減。ユーザー情報の自動入力でフォーム完了率が高い。リードの質はウェブサイトフォームより低い傾向。

**リード品質を高めるコツ:** フォームにカスタム質問追加、「Higher Intent」フォームタイプ選択、サンクス画面でCTA設定

**選ぶべきケース:** メルマガ登録、資料請求、問い合わせ、モバイルでのフォーム離脱が課題の場合

### 2-6. Instant Experience（旧Canvas）

**概要:** 広告タップ時にフルスクリーンで展開されるインタラクティブな体験。

**選ぶべきケース:** ブランド体験を没入的に伝えたい場合、商品カタログの詳細閲覧、モバイル向けLP代わり

### 2-7. フォーマット選定マトリクス

| 目的 | 第1推奨 | 第2推奨 | 第3推奨 | 理由 |
|------|--------|--------|--------|------|
| **Awareness** | 動画/リールズ | カルーセル | 画像 | 動画が最も注目を集める |
| **Traffic** | 画像 | カルーセル | 動画 | 明確なCTAでクリック促進 |
| **Engagement** | 動画/リールズ | カルーセル | 画像 | インタラクション促進 |
| **Leads** | リード広告 | 動画 | 画像 | フォーム内完結で離脱低減 |
| **Sales（EC）** | コレクション | カルーセル | 動的商品広告 | 商品ブラウジング＋購入導線 |
| **Sales（非EC）** | 動画 | 画像 | カルーセル | 訴求力＋CTA |
| **App Promotion** | 動画 | プレイアブル | 画像 | アプリ体験の視覚化 |

## 3. 広告配置面（Placements）

### 3-1. 全配置面の一覧

#### Facebook の配置面

| 配置面 | 説明 | 対応フォーマット | パフォーマンス特性 |
|--------|------|-----------------|-------------------|
| **Feed** | メインのニュースフィード | 画像、動画、カルーセル、コレクション | リーチ最大。ほぼ全DAUにリーチ可能 |
| **Marketplace** | Marketplace の閲覧中に表示 | 画像、動画、カルーセル | 購買意向の高いユーザーにリーチ |
| **Video Feeds** | 動画フィード（Facebook Watch含む） | 動画 | 動画視聴に集中しているユーザー |
| **Right Column** | デスクトップの右カラム | 画像 | CPC最低だがリーチ限定。リターゲティング向き |
| **Stories** | ストーリーズ間に表示 | 画像、動画 | フルスクリーン・没入型 |
| **Reels** | リールズ間に表示 | 動画 | 高エンゲージメント、低CPM |
| **In-Stream Video** | 動画コンテンツの再生中に表示 | 動画 | 動画視聴中の集中したユーザー |
| **Search Results** | 検索結果に表示 | 画像、動画 | 能動的に情報を探しているユーザー |
| **Business Explore** | ビジネス投稿の閲覧時に表示 | 画像、動画 | ビジネスコンテンツに関心あるユーザー |
| **Groups Feed** | グループフィード内に表示 | 画像、動画 | コミュニティ内のリーチ |

#### Instagram の配置面

| 配置面 | 説明 | 対応フォーマット | パフォーマンス特性 |
|--------|------|-----------------|-------------------|
| **Feed** | メインフィード | 画像、動画、カルーセル、コレクション | Facebookの最大10倍のエンゲージメント |
| **Stories** | ストーリーズ間に表示 | 画像、動画 | CPC $1.12（Feed $1.58 より低い）。没入型 |
| **Reels** | リールズ間に表示 | 動画 | 最も低いCPM。ROAS +13%、CPA -16% |
| **Explore Feed** | Explore内のコンテンツをタップ後に表示 | 画像、動画 | 新規発見志向のユーザー。低競争・低CPM |
| **Explore Home** | Exploreのグリッドタイルとして表示 | 画像、動画 | 発見フェーズのユーザーにリーチ |
| **Shop** | Instagram Shop内に表示 | 画像 | 購買意向の高いユーザー |
| **Search Results** | 検索結果に表示 | 画像、動画 | キーワード検索ユーザーにリーチ |

#### Messenger の配置面

| 配置面 | 説明 | 対応フォーマット | パフォーマンス特性 |
|--------|------|-----------------|-------------------|
| **Home Screen** | チャットスレッド間に表示 | 画像 | メッセージ確認時にリーチ |
| **Sponsored Messages** | チャット内にメッセージとして表示 | テキスト＋画像 | 既存接触ユーザーへの直接リーチ |

#### Audience Network / Threads

| 配置面 | 説明 | 対応フォーマット | パフォーマンス特性 |
|--------|------|-----------------|-------------------|
| **Native/Banner/Interstitial** | Meta外のアプリ/サイトに表示 | 画像、動画 | 低CPMだがCV意向は低い。認知向き |
| **Rewarded Video** | アプリ内のリワード動画 | 動画 | 視聴完了率が高い |
| **Threads Home Feed** | Threadsフィード内に表示 | 画像、動画 | 2025年に新規追加。リーチ拡大に有効 |

### 3-2. 配置面の品質ティア

| ティア | 配置面 | 特徴 | 推奨用途 |
|--------|--------|------|---------|
| **高品質** | Facebook/Instagram Feed、Reels、Stories | CV獲得力が高い、エンゲージメントが高い | 全目的、特にSales/Leads |
| **中品質** | Instagram Explore、Facebook Marketplace、In-Stream Video | 中程度のリーチ | Awareness/Traffic/Sales |
| **低品質** | Audience Network、Messenger、Right Column | インプレッション重視 | Awareness/リーチ拡大のみ |

### 3-3. Advantage+ Placements vs 手動配置

| 項目 | Advantage+ Placements | 手動配置 |
|------|----------------------|---------|
| 仕組み | AIが全配置面から最適な配信先を自動選択 | 広告主が配信先を個別に選択 |
| リーチ | 最大6%多くリーチ（予算増なし） | 選択した配置面のみ |
| CPM | 最大22%低下 | 配置面による |
| CPA | 最大32%低下（Meta公式） | コントロール可能 |
| 推奨ケース | **CV目的（Sales/Leads/App）— デフォルト推奨** | Traffic/Impression目的、特定配置面でのテスト |

**基本方針:** CV目的のキャンペーンは **Advantage+ Placements がデフォルト推奨**。

### 3-4. 配置面別のクリエイティブ仕様

#### アスペクト比

| 配置面 | 推奨アスペクト比 | 解像度 |
|--------|----------------|--------|
| Feed（Facebook/Instagram） | 4:5（最推奨）、1:1 | 1080×1350、1080×1080 |
| Stories（Facebook/Instagram） | 9:16 | 1080×1920 |
| Reels（Facebook/Instagram） | 9:16 | 1080×1920 |
| In-Stream Video | 16:9 | 1920×1080 |
| Right Column | 1:1 | 1080×1080 |
| Marketplace | 1:1 | 1080×1080 |
| Messenger | 1.91:1 | 1200×628 |

#### セーフゾーン

| 配置面 | 上部マージン | 下部マージン | 左右マージン | セーフ領域 |
|--------|------------|------------|------------|-----------|
| **Feed（1:1）** | 100px（10%） | 100px（10%） | 100px（10%） | 中央80%以内 |
| **Feed（4:5）** | 250px | 250px | — | 中央部 |
| **Stories** | 270px（14%） | 380px（20%） | 65px（6%） | 中央 950×1420px |
| **Reels** | 270px（14%） | **670px（35%）** | 65px（6%） | 中央部（下部が特に大きい） |

> **設計の鉄則:** 9:16で設計し、Reelsのセーフゾーン（下部35%）に収まるようにすれば、Stories含む全配置面で安全に表示される。

### 3-5. ビジネスタイプ別の推奨配置戦略

| ビジネスタイプ | 推奨配置 | 理由 |
|---------------|---------|------|
| **EC / D2C** | Advantage+（全自動）、Instagram Feed + Reels 重視 | 商品ビジュアル × 高エンゲージメント |
| **B2Bリード獲得** | Facebook Feed + Instagram Feed | ビジネス層のリーチ。Audience Network は除外検討 |
| **アプリ** | Advantage+（全自動） | AI がインストール確率の高い配置面を最適化 |
| **ローカルビジネス** | Facebook Feed + Marketplace + Instagram Feed | 地域ターゲティングとの組み合わせ |
| **ブランド認知** | Advantage+（全自動）、動画はReels + Stories優先 | 最大リーチ × 没入型体験 |
| **コンテンツマーケティング** | Facebook Feed + Instagram Feed + Explore | コンテンツ発見と拡散 |

## 4. 2025年ベンチマーク（参考値）

| 指標 | Facebook平均 | Instagram平均 | 備考 |
|------|------------|-------------|------|
| CPC | $1.31 | $1.42 | Instagram Stories: $1.12 / Feed: $1.58 |
| CPM | $15.26 | $9.68 | Reels が最もCPM低い |
| CTR | 0.90% | 0.22〜0.88% | 業界・フォーマットにより大きく変動 |
| CPL | $27.66 | — | 前年比約20%上昇、それでもGoogle Adsより安い |

> 注: ベンチマークは業界・地域・季節により大幅に変動する。あくまで初期設計の参考値として使用する。
