## 1. 推奨キャンペーン構造

### 基本方針: シンプルな構造

2026年のMeta広告は、複雑なキャンペーン階層から**シンプルな2〜3キャンペーン構造**へ移行している。Andromedaアルゴリズムにより、シンプルな構造の方がAIの学習サイクルが速く、パフォーマンスが向上する。

### パターン A: 2キャンペーン構造（最もシンプル）

```
広告アカウント
├── 1. Creative Testing Campaign（クリエイティブテスト）
│   ├── 予算: 全体の10〜20%
│   ├── 目的: 新しいクリエイティブの発見
│   ├── 設定: CBO、ブロードターゲティング
│   └── 広告セット: 1つ（複数コンセプトを1セットに投入）
│
└── 2. Winning Ads Campaign（スケーリング）
    ├── 予算: 全体の80〜90%
    ├── 目的: 検証済みクリエイティブのスケール
    ├── 設定: CBO、ブロードターゲティング
    └── 広告セット: 勝ちクリエイティブを集約
```

### パターン B: 3キャンペーン構造（フルファネル）

```
広告アカウント
├── 1. Sales Campaign（CBO / Advantage+）
│   ├── 予算配分: 最大
│   ├── 目的: フルファネルでの売上/CV獲得
│   ├── 設定: Advantage+ Sales Campaign、ブロードターゲティング
│   └── 広告セット: 勝ちクリエイティブを配置
│
├── 2. Awareness Campaign
│   ├── 予算配分: 中
│   ├── 目的: 新規ユーザーへの認知獲得
│   ├── 最適化: 動画再生（ThruPlay）またはリーチ
│   └── 広告セット: 動画コンテンツ中心
│
└── 3. Remarketing Campaign
    ├── 予算配分: 小
    ├── 目的: 既存接点ユーザーの購買転換
    ├── 設定: ABO（予算コントロール重視）
    └── 広告セット: カスタムオーディエンス別
```

### リターゲティングの扱い

2026年では、MetaのAIが自動的にリターゲティングを処理するようになっている:

- Advantage+ Sales Campaign は予算の20〜30%を自動的にリターゲティングに配分
- 多くの広告主が**専用のリターゲティングキャンペーンを廃止**し、メインキャンペーンに統合
- ただし、特定のカスタムオーディエンスに対して異なるメッセージを出したい場合は、別キャンペーンが有効

## 2. 予算配分の設計

### ファネル別予算配分

| 段階 | 予算配分 | 目的 |
|------|---------|------|
| 認知・検討（ToF + MoF） | 60% | 新規オーディエンスの獲得、パイプライン構築 |
| 購入転換（BoF） | 40% | CV獲得、直接的な売上 |

### テスト vs スケール

| 用途 | 予算配分 |
|------|---------|
| クリエイティブテスト | 10〜20% |
| スケーリング（勝ちクリエイティブ） | 80〜90% |

### 予算スケーリングのルール

- **週20〜30%** の予算増加が上限（CACが維持できる場合のみ）
- 急激な予算増加はAIの学習をリセットし、パフォーマンスが悪化する
- スケールは **個別の広告ではなく、コンセプト単位** で判断する
- 予算増加後も数日間はパフォーマンスを注視する

## 3. フルファネル戦略の設計

### ファネル各段階の設計

#### Stage 1: 認知（Awareness）

| 項目 | 設定 |
|------|------|
| 目的 | Awareness |
| ターゲティング | ブロードオーディエンス |
| 推奨オーディエンスサイズ | 100K〜500K |
| クリエイティブ | 動画/リールズ中心、ブランドストーリー |
| 最適化 | リーチ、ThruPlay |
| KPI | CPM、動画視聴率、リーチ |

#### Stage 2: 検討（Consideration）

| 項目 | 設定 |
|------|------|
| 目的 | Traffic / Engagement |
| ターゲティング | 動画視聴者、エンゲージメントオーディエンス |
| クリエイティブ | カルーセル、動画、テスティモニアル |
| 最適化 | ランディングページビュー、動画視聴 |
| KPI | CTR、ランディングページビュー、エンゲージメント率 |

#### Stage 3: 購入転換（Conversion）

| 項目 | 設定 |
|------|------|
| 目的 | Sales / Leads |
| ターゲティング | サイト訪問者、カート放棄者、リード |
| クリエイティブ | 動的商品広告、コレクション、オファー訴求 |
| 最適化 | 購入、リード獲得 |
| KPI | CPA、ROAS、CV数、CVR |

### 除外設定の重要性

- プロスペクティングキャンペーンから**既存顧客と最近のエンゲージユーザーを除外**
- 予算の無駄遣いを防止
- 各ファネル段階で異なるメッセージを届けるために必須

## 4. 学習フェーズの管理

### 学習フェーズとは

MetaのAIがキャンペーンの最適なオーディエンス、配置、入札を発見するための初期期間。

### 学習フェーズの要件

- 通常 **7日間** で完了（50件以上の最適化イベントが必要）
- 学習中はパフォーマンスが不安定になるのは正常
- 学習フェーズ中は**大きな変更を避ける**

### 学習フェーズをリセットする変更

以下の変更は学習フェーズをリセットするため、頻繁に行わない:

- ターゲティングの変更
- 入札戦略の変更
- 予算の大幅変更（20%以上）
- 広告の一時停止/再開
- 新しい広告の追加
- 広告クリエイティブの変更

### 学習フェーズを成功させるコツ

- 十分な予算を確保（CPA目標 × 50件/週 以上）
- 最適化イベントは実際に発生頻度の高いものを選択
- CV数が少ない場合は、ファネルの上位イベント（カート追加など）で最適化を検討

## 5. パフォーマンス計測と最適化

### 主要KPI

| KPI | 目的 | 目安 |
|-----|------|------|
| CPA（獲得単価） | CV効率の評価 | 商品/サービスの利益から逆算 |
| ROAS（広告費用対効果） | 売上効率の評価 | 業界・商品により異なる |
| CTR（クリック率） | クリエイティブの訴求力 | 1%以上が目安 |
| CPM（1000imp単価） | リーチ効率 | 業界・ターゲットにより変動 |
| CPMr（1000リーチ単価） | オーディエンス疲労の指標 | $20以下が健全 |
| フリクエンシー | 広告疲労の指標 | 3.0以上は要注意 |
| CVR（コンバージョン率） | LP/ファネルの効率 | 1〜3%が一般的目安 |
| LTV:CAC比率 | 長期的な収益性 | 3:1以上が理想 |

### 最適化のアクション

| 状況 | アクション |
|------|-----------|
| CPMr上昇 | クリエイティブをリフレッシュ（入札調整ではない） |
| フリクエンシー3.0超 | 新しいクリエイティブを投入、オーディエンス拡大 |
| CTR低下 | フックとクリエイティブコンセプトの見直し |
| CPA上昇 | クリエイティブ、LP、オーディエンスの順で見直し |
| CVR低下（LP側） | LP の改善が必要（広告の問題ではない） |

### アトリビューション設定

| ウィンドウ | 内容 | 推奨ケース |
|-----------|------|-----------|
| 7日クリック + 1日ビュー | デフォルト | ほとんどの場合に推奨 |
| 1日クリック | 厳密なアトリビューション | 検証目的、低単価商品 |
| 28日クリック | 長い検討期間の商品 | 高額商品、B2B |

## 6. 避けるべきミス

### 構造に関するミス

- **キャンペーンの乱立:** 少数のキャンペーンに統合すべき。AIの学習データが分散する
- **オーディエンスの重複:** 同じユーザーに複数キャンペーンから配信すると、自社内オークション競争が発生
- **頻繁な構造変更:** 学習フェーズが繰り返しリセットされ、安定しない

### クリエイティブに関するミス

- **フォーマットの固定化:** 1つのフォーマットに依存すると広告疲労が早い
- **テスト不足:** 推測ではなくデータに基づく判断が必要
- **配置面に合わないクリエイティブ:** アスペクト比やスタイルを配置面ごとに最適化

### 設定に関するミス

- **Pixel/CAPI未設置での出稿:** 最適化の基盤がない状態では効果的な運用は不可能
- **十分なCVデータなしでのASC利用:** 学習に必要なデータがないと効果が出ない
- **予算の急激な変更:** AIの学習をリセットしてしまう
- **自動クリエイティブ機能への過信:** 自動生成された組み合わせのモニタリングは必須

## 7. ビジネスタイプ別の推奨設計まとめ

### EC / D2C

| 項目 | 推奨設定 |
|------|---------|
| キャンペーン目的 | **Sales**（ASC推奨） |
| キャンペーン構造 | パターンA（2CP: テスト + スケーリング） |
| 最適化イベント | Purchase（不足時はAdd to Cart） |
| フォーマット | コレクション、カルーセル、動的商品広告、動画 |
| 配置面 | **Advantage+（全自動）** — Instagram Feed + Reels 重視 |
| ターゲティング | ブロード + カスタムオーディエンス（購入者）のLookalike |
| 入札戦略 | 最低コスト → ROAS目標 |
| 予算配分 | テスト10〜20% / スケール80〜90% |

### B2B リード獲得

| 項目 | 推奨設定 |
|------|---------|
| キャンペーン目的 | **Leads** または **Sales** |
| キャンペーン構造 | パターンB（3CP: Sales + Awareness + Remarketing） |
| 最適化イベント | Lead / Form Submit |
| フォーマット | リード広告（Instant Form）、動画、画像 |
| 配置面 | Facebook Feed + Instagram Feed（Audience Network は除外検討） |
| ターゲティング | ブロード + 顧客リストLookalike、ウェブサイト訪問者RT |
| 入札戦略 | コストキャップ |
| 備考 | Instant Form vs ウェブサイトCVの比較テスト推奨 |

### SaaS / サブスクリプション

| 項目 | 推奨設定 |
|------|---------|
| キャンペーン目的 | **Leads** → **Sales** |
| キャンペーン構造 | パターンB（3CP） |
| 最適化イベント | Demo Request / Sign Up / Trial Start |
| フォーマット | 動画（デモ）、カルーセル（機能紹介）、画像 |
| 配置面 | Facebook Feed + Instagram Feed |
| ターゲティング | ブロード + カスタムオーディエンス（サイト訪問者、動画視聴者） |
| 入札戦略 | 最低コスト → コストキャップ |
| 備考 | 無料トライアル訴求が最も効果的 |

### ローカルビジネス（店舗・サロン・クリニック等）

| 項目 | 推奨設定 |
|------|---------|
| キャンペーン目的 | **Awareness** + **Sales** |
| キャンペーン構造 | パターンB（Awareness + Sales + Remarketing） |
| 最適化イベント | Reach（認知）/ Purchase or Lead（CV） |
| フォーマット | 動画（店内ツアー）、画像（クーポン）、カルーセル |
| 配置面 | Facebook Feed + Marketplace + Instagram Feed（地域ターゲティング） |
| ターゲティング | 地域ターゲティング + ブロード |
| 入札戦略 | 最低コスト |
| 備考 | 地域を絞った配信が重要 |

### アプリビジネス

| 項目 | 推奨設定 |
|------|---------|
| キャンペーン目的 | **App Promotion** |
| キャンペーン構造 | パターンA（テスト + スケーリング） |
| 最適化イベント | Install → In-App Event → Value |
| フォーマット | 動画（アプリデモ）、プレイアブル広告 |
| 配置面 | **Advantage+（全自動）** |
| ターゲティング | ブロード + 課金ユーザーLookalike |
| 入札戦略 | 最低コスト → バリュー最適化 |
| 備考 | 50件以上インストール後に高価値イベントで最適化に移行 |

### ブランド認知（大手/ブランドビルディング）

| 項目 | 推奨設定 |
|------|---------|
| キャンペーン目的 | **Awareness** |
| キャンペーン構造 | 単一Awarenessキャンペーン |
| 最適化イベント | Reach / Ad Recall Lift / ThruPlay |
| フォーマット | 動画/リールズ（最も効果的）、画像 |
| 配置面 | **Advantage+（全自動）** — Reels + Stories 優先 |
| ターゲティング | ブロード（最大リーチ） |
| 入札戦略 | 最低コスト |
| 備考 | 動画中心のブランドストーリーテリング |

## 8. 命名規則

### キャンペーン名

**フォーマット:** `{目的}_{ターゲット}_{補足}`

例:
- `Sales_Prospecting_ASC`
- `Sales_Retargeting_CartAbandoners`
- `Awareness_Video_Broad`
- `Leads_InstantForm_Japan`
- `CreativeTest_Weekly`

### 広告セット名

**フォーマット:** `{オーディエンス}_{配置}_{補足}`

例:
- `Broad_AllPlacements`
- `LAL1pct_Purchase_Feed`
- `Retargeting_WebVisitors30d`
- `CustomAudience_EmailList`

### 広告名

**フォーマット:** `{フォーマット}_{訴求}_{バリエーション}`

例:
- `Video_Testimonial_v1`
- `Image_PainPoint_4x5`
- `Carousel_ProductLine_v2`
- `UGC_FounderStory_Reel`

### ルール

- 区切り文字はアンダースコア `_` で統一
- 英語ベース推奨
- 日付はテスト施策のみ `_YYMM` を末尾に付与
- 大文字始まりのキャメルケースで単語を区切る
