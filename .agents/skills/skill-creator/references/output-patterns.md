# 出力パターン

スキルが一貫した高品質の出力を生成する必要がある場合に、これらのパターンを使用すること。

## テンプレートパターン

出力形式のテンプレートを提供する。厳密さのレベルはニーズに合わせること。

**厳密な要件の場合（APIレスポンスやデータ形式など）：**

```markdown
## レポート構成

必ずこの正確なテンプレート構成を使用すること：

# [分析タイトル]

## エグゼクティブサマリー
[主要な発見事項の概要を1段落で]

## 主要な発見事項
- 裏付けデータ付きの発見事項1
- 裏付けデータ付きの発見事項2
- 裏付けデータ付きの発見事項3

## 推奨事項
1. 具体的で実行可能な推奨事項
2. 具体的で実行可能な推奨事項
```

**柔軟なガイダンスの場合（適応が有用な場合）：**

```markdown
## レポート構成

以下は妥当なデフォルト形式だが、最善の判断で調整すること：

# [分析タイトル]

## エグゼクティブサマリー
[概要]

## 主要な発見事項
[発見した内容に基づいてセクションを調整]

## 推奨事項
[特定のコンテキストに合わせて調整]

特定の分析タイプに応じて、必要に応じてセクションを調整すること。
```

## 例示パターン

出力品質が例を見ることに依存するスキルでは、入力/出力のペアを提供する：

```markdown
## コミットメッセージの形式

以下の例に従ってコミットメッセージを生成すること：

**例1：**
入力：JWTトークンによるユーザー認証を追加
出力：
```
feat(auth): implement JWT-based authentication

Add login endpoint and token validation middleware
```

**例2：**
入力：レポートで日付が正しく表示されないバグを修正
出力：
```
fix(reports): correct date formatting in timezone conversion

Use UTC timestamps consistently across report generation
```

このスタイルに従うこと：type(scope): 簡潔な説明、その後に詳細な説明。
```

例は、説明文だけよりも、Claudeが望ましいスタイルと詳細レベルをより明確に理解するのに役立つ。
